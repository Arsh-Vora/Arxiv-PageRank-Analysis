## 📌 Overview
This repository contains the code and research for my Master's project in the **Algorithms for Massive Datasets** module. The goal of this project was to implement the PageRank algorithm on a massive co-authorship network to identify the most influential "hubs" in the academic world. 

Processing the **ArXiv Metadata Snapshot** (1.96 million unique authors and 2.97 million papers) presented severe computational challenges. This project explores the engineering trade-offs between standard graph processing, bipartite graph restructuring, and state-of-the-art matrix approximations to navigate memory limits and Data Skew.

## 🛑 The Challenge: "The Unipartite Nightmare"
Naively mapping standard co-authorships (Author $\to$ Author) creates a fully connected clique for every paper. The number of directed edges $E$ generated by a single paper with $n$ authors is:
$E = n(n-1)$

High-Energy Physics collaborations (e.g., ATLAS, CMS) routinely feature 3,000+ authors. A single paper generates **~9 million edges**, instantly causing Apache Spark's JVM to throw Out-Of-Memory (OOM) errors during the shuffle phase due to severe Data Skew.



## 🛠️ Implemented Solutions
To solve this, I developed and benchmarked three distinct methodologies:

### 1. Filtered Spark Map-Reduce (The Compromise)
* **Method:** Standard PySpark RDD Map-Reduce.
* **Optimization:** Applied a "Survival Filter" to discard papers with $> 50$ authors to prevent memory crashes.
* **Trade-off:** Successfully executed in ~90 minutes but introduced academic bias by artificially erasing massive physics collaborations.

### 2. Bipartite Spark Graph (The Ground Truth)
* **Method:** Reorganized the topology into a Bipartite Graph (Author $\to$ Paper $\to$ Author). 
* **Optimization:** Bounded edge growth to $O(N)$ (a 3,000-author paper generates exactly 6,000 edges instead of 9 million). 
* **Result:** Successfully processed 100% of the unfiltered dataset in ~50 minutes, proving that architectural shifts solve Big Data bottlenecks better than data deletion.



### 3. $T^2$-Approx Matrix Approximation (The Baseline)
* **Method:** Implemented a novel low-rank matrix approximation algorithm using `scipy.sparse`.
* **Optimization:** Utilized Degree-Weighted Sampling to compress the graph into matrices $X$ and $Y$, performing power iteration on a much smaller matrix $Z = Y \cdot X$.
* **Result:** Achieved convergence in just **84.69 seconds**, bypassing PySpark's distributed shuffle overhead entirely.

## 📊 Key Findings
* **Entity Resolution is Critical:** The filtered Spark run identified "Liu, Yang" as the #1 author. However, this exposed a data ambiguity issue: Spark merged hundreds of distinct researchers with the same name into a single "Super-Node".
* **Big Science Dominates:** The Bipartite run correctly identified the **CMS Collaboration** and **ATLAS Collaboration** as the true hubs of the network, accurately reflecting the highly interconnected nature of modern physics.
